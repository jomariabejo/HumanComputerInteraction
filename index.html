EMPLOYEE_ATTENDANCE_NAV_SELECTED

In the attendance section, we provide a table that shows a user attendance history. User can use this table to look at their previous attendance records and choose a specific time period they want to review. We also include a text field where user can type in a month or year. It's worth noting that the table will be empty if no information is entered into the text field; in other words, it only displays data when you provide input. We've included a pagination feature, although it currently doesn't show separate pages because creating new pages is a bit challenging. However, you can assume that this button will become clickable. When you click on the pagination, the table will also update according to the page number you select.

EMPLOYEE_LEAVE_NAV_SELECTED

Here, you can view the remaining leave credits for the employee, allowing them to check their available leave days, the leave days they've already used for specific types of leave, and their remaining balance.


Additionally, there's a table displaying the leave requests made by employees, and you can see that some of them have already been approved. When a new leave request is successfully submitted, the leave credits will be updated, and the request will initially be marked as pending. Only the admin or superuser has the authority to approve employee leave requests.


To make it more user-friendly, we've included a pagination feature. This is because in some cases, employees may have a large amount of data. Adding pagination helps efficiently manage and query this data.

EMPLOYEE_LEAVE_NAV_SET_LEAVE_SELECTED

We have four distinct components, and for the submit button to become enabled or clickable, all of these components must have valid input. This requirement is in place to prevent user errors, enhance system efficiency, and ensure consistency in the data received by the system.
